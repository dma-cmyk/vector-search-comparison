# ベクトル検索 比較ラボ (Vector Search Comparison Lab)

Gemini API の Embeddings を活用し、異なるデータ構造（「要約のみ」vs「要約＋タグ」）に基づくベクトル検索の精度や特性を比較・検証するための React アプリケーションです。

## 📖 概要

このアプリケーションは、ウェブサイト情報の検索において、どのようなメタデータ（要約、タグ、カテゴリなど）をベクトル化の対象に含めるかで検索結果がどのように変化するかを視覚的に比較することを目的としています。
ブラウザ上の IndexedDB にベクトルデータを保存するため、サーバーレスで動作します。

## ✨ 主な機能

*   **2種類のベクトル検索比較:**
    *   **要約マッチ:** コンテンツの「要約」のみをベクトル化して検索。
    *   **要約＋タグ マッチ:** 「要約」と「タグ」を組み合わせたテキストをベクトル化して検索。
    *   これらを並べて表示し、検索意図に対する応答の違いを比較できます。
*   **Gemini API 連携:**
    *   Google の Gemini モデル（Embedding および Text Generation）を直接利用します。
    *   利用可能なモデルを動的に取得・選択可能です。
*   **URL / 画像 解析・自動登録:**
    *   任意の URL や画像をアップロードすると、Gemini が自動的に「要約」「カテゴリ分類」「タグ付け」を行い、インデックスに追加します。
*   **ブラウザ内データ管理:**
    *   IndexedDB を使用してデータを永続化。
    *   事前定義された50件のサンプルデータ（テック、料理、旅行など10カテゴリ）をワンクリックでインデックス化可能。
    *   データの閲覧、個別削除、JSONエクスポート機能。

## 🛠️ 技術スタック

*   **Frontend:** React, Tailwind CSS
*   **AI / API:** Google Gemini API (Generative Language API)
*   **Database:** IndexedDB (Browser Native)
*   **Icons:** Lucide React

## 🚀 使い方

### 1. 準備
このアプリケーションを使用するには、Google AI Studio で取得した **Gemini API キー** が必要です。

### 2. アプリケーションの起動
```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動
npm start
```

### 3. 操作フロー
1.  画面右上の入力欄に **Gemini API キー** を入力します。
2.  **「ライブラリ」** タブを開き、「一括インデックス構築を実行」ボタンを押します。
    *   内蔵されている50件のサイトデータがベクトル化され、データベースに保存されます。
3.  **「検索比較」** タブで検索ワード（例：「AIの最新トレンド」「週末に行ける温泉」）を入力し、検索します。
    *   左右に異なるロジックでの検索結果が表示されます。
4.  **「URL追加」** タブでは、気になるWeb記事のURLを入力するか画像をアップロードすることで、AIが内容を解析し、検索対象に追加できます。

## ⚠️ 注意事項
*   APIキーはブラウザ内にのみ保存（React state）され、外部サーバーには送信されませんが、取り扱いには十分ご注意ください。
*   大量のデータを一気にインデックス化する場合、API のレート制限（Rate Limit）にかかる可能性があります。
