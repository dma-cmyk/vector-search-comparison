# ベクトル検索 比較ラボ (Vector Search Comparison Lab)

Google Gemini API の Embeddings を活用し、検索ロジックの違い（「要約のみ」vs「要約＋タグ」）が検索結果にどう影響するかを視覚的に比較・実験できる React アプリケーションです。

[🚀 **デモサイトを開く**](https://dma-cmyk.github.io/vector-search-comparison/)

## 📖 概要

検索システムを作る際、「何をベクトル化して検索対象にするか」は精度に大きく関わります。このツールでは、同じデータに対して以下の2パターンの検索を同時に行い、結果を左右に並べて比較できます。

1.  **要約マッチ:** コンテンツの「要約文」だけをベクトル化して検索（文脈重視）
2.  **要約＋タグ マッチ:** 「要約文」と「キーワードタグ」を混ぜてベクトル化して検索（キーワード重視）

## ✨ 主な機能

*   **リアルタイム比較検索:** 検索ワードを入力すると、2つの異なるロジックでの検索結果を瞬時に比較表示。
*   **サーバーレス構成:** データはすべてブラウザ内のデータベース（IndexedDB）に保存。外部サーバーへのデータ送信はありません。
*   **サンプルデータ内蔵:** 50件の多様なWebサイトデータ（テック、料理、旅行など）をワンクリックでインデックス化。
*   **AIによる自動解析:** 任意のURLや画像を渡すと、Geminiが自動で「要約・カテゴリ分類・タグ付け」を行い、データベースに追加できます。

## 🚀 使い方（デモサイトで試す）

特別なインストールは不要です。以下のステップですぐに試せます。

### ステップ 1: APIキーの準備
このアプリは Google の AI モデルを利用するため、Gemini API キーが必要です。
*   まだ持っていない場合は [Google AI Studio](https://aistudio.google.com/app/apikey) で無料で取得してください。

### ステップ 2: 初期設定
1.  [デモサイト](https://dma-cmyk.github.io/vector-search-comparison/) にアクセスします。
2.  画面右上の入力欄に **Gemini API キー** を入力します。
    *   *※キーはブラウザ内にのみ保存され、外部には送信されません。*

### ステップ 3: データの準備（インデックス構築）
1.  メニューから **「ライブラリ」** タブをクリックします。
2.  **「一括インデックス構築を実行」** ボタンを押します。
3.  数秒〜数十秒待つと、内蔵された50件のデータがベクトル化され、検索できる状態になります。

### ステップ 4: 検索の実験
1.  メニューから **「検索比較」** タブに戻ります。
2.  検索ボックスに質問を入力してください。
    *   例: 「週末に行ける温泉旅行」
    *   例: 「初心者向けのプログラミング学習」
    *   例: 「最新のAIトレンド」
3.  左右の検索結果を見比べて、精度の違いを楽しんでください。

### ステップ 5: 独自データの追加（任意）
1.  **「URL追加」** タブを開きます。
2.  気になるWeb記事のURLを入力するか、画像をアップロードします。
3.  「AI解析してインデックスに追加」を押すと、その記事が検索対象に追加されます。

## 💻 ローカルで開発・実行する場合

自分のPCでコードを動かしたり改造したい場合は、以下の手順で行ってください。

### 必須要件
*   Node.js (v16以上推奨)
*   npm

### インストールと起動

```bash
# リポジトリをクローン
git clone https://github.com/dma-cmyk/vector-search-comparison.git
cd vector-search-comparison

# 依存パッケージをインストール
npm install

# 開発サーバーを起動
npm run dev
```

ブラウザで `http://localhost:5173` が自動的に開きます。

## 🛠️ 技術スタック

*   **Frontend:** React, Tailwind CSS, Vite
*   **AI / LLM:** Google Gemini API (gemini-2.5-flash-preview等)
*   **Vector Search:** Client-side Cosine Similarity
*   **Database:** IndexedDB (idb)
*   **Hosting:** GitHub Pages
